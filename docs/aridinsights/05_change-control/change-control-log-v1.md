# Change Control Log v1

## Purpose
This log explains what changed, why, and what it impacts. It protects you when:
- a customer questions an output
- a partner audits your data lineage
- you debug regressions

## Change Entry Template
- Change ID:
- Date:
- Change Type: SCHEMA | PROMPT | DATA_SOURCE | THRESHOLD | PRODUCT_COPY | UI
- Summary:
- Rationale:
- Impacted Products: SCORECARD | AUDIT | MONITORING | NAVIGATOR | CONTENT
- Impacted Geos: FL-ORLANDO | FL-TAMPA | OTHER
- Risk Level: LOW | MED | HIGH
- Rollback Plan:
- Validation Steps:
- Evidence Links (internal refs):
- Approved By:

## Required attachments for HIGH risk changes
- before/after prompt diff
- sample context pack
- sample evidence log
- output comparison notes

## Non-negotiable
If a change affects thresholds or evidence rules, you must:
- update AIG-OS Constitution
- update Risk & Disclaimers (if language changes)
- update Coverage Ledger (if coverage status changes)
